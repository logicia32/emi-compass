# emi-compass

EMI 対策部品 (コンデンサ / 三端子コンデンサ / チップビーズ) の「効き」を、
等価回路モデルからグラフにして読めるようにする小さな道具です。

データシートに載っている「インピーダンス |Z| vs 周波数」や「挿入損失」のグラフを
**自分の手で描いて**、部品を選ぶ前に挙動を確かめることを狙っています。
実験設備も有償ツールも不要、Python だけで動きます。

Zenn の連作「**EMI 対策部品の羅針盤**」の伴走ツールです。

| 連作 | 内容 | このリポジトリでの対応 |
|---|---|---|
| [第 1 本 (コンデンサ編)](https://zenn.dev/logicia32/articles/2026-06-11-emi-compass-1-capacitor) | 「0.1μF」はいつまでコンデンサか。V 字カーブ・SRF・並列反共振・三端子 | `cap` サブコマンド (実装済み) |
| [第 2 本 (チップビーズ編)](https://zenn.dev/logicia32/articles/2026-06-12-emi-compass-2-bead) | ビーズは高周波で「抵抗になる」部品。とりあえずビーズの落とし穴 | `bead` サブコマンド (実装済み) |
| 第 3 本 (羅針盤編) | 場面別の選定フロー。メーカー配布の実部品データで買う前に検証 | データ読み込みと組み合わせ検証 (予定) |

## これは何を解決する道具か

ノイズ対策部品は「あってもなくても回路が動く」ので、効果が直感的に見えません。
効果は**周波数軸の上にしか現れない**からです。そして、その答えはデータシートの
インピーダンスグラフに最初から全部描いてあります。

このツールは、そのグラフを教科書的な等価回路 (実コンデンサ = C + ESL + ESR の直列)
から描き起こします。たとえば:

- 0.1μF (ESL 0.5nH) が「コンデンサとして」働けるのは **自己共振周波数 (SRF) ≈ 22.5 MHz まで**。それより上ではほぼインダクタ
- 容量を 0.01μF → 1μF に 100 倍しても、**SRF より上の |Z| は 1 本も動かない** (右斜面は ESL が決めるため)
- 0.1μF と 0.01μF を並列にすると、谷 2 つの間 (≈ 53 MHz) に**反共振の山** (|Z| が ESR の 20 倍超) が立つ
- 同じ 0.1μF でも、三端子コンデンサ (ESL 1/10) なら **100 MHz の |Z| が約 12 分の 1**
- チップビーズは「100Ω@100MHz」の 1 点では選べない。R が X を追い越す **目覚めの周波数 (このモデルで ≈ 51 MHz)** より上で初めてノイズを熱に変える

…という「先輩の知恵の数値的な根拠」が、コマンド一発で全部グラフになります。

## インストール

Python 3.11+ が必要です。

```bash
pip install git+https://github.com/logicia32/emi-compass
```

開発用 (clone して使う):

```bash
git clone https://github.com/logicia32/emi-compass
cd emi-compass
python -m venv .venv
.venv/bin/pip install -e ".[dev]"
```

## 使い方

### 実コンデンサ 1 個の V 字カーブと SRF

```bash
emi-compass cap --c 0.1u --esl 0.5n --esr 0.02
```

```
C1: C=0.1 uF, ESL=0.5 nH, ESR=0.02 Ohm
    SRF = 22.5 MHz  (|Z| there ~= ESR)
saved: emi_compass_cap.png
```

### 容量違い 2 個並列の反共振チェック

```bash
emi-compass cap --c 0.1u --c2 0.01u
```

```
C1: C=0.1 uF, ESL=0.5 nH, ESR=0.02 Ohm
    SRF = 22.5 MHz  (|Z| there ~= ESR)
C2: C=0.01 uF, ESL=0.5 nH, ESR=0.02 Ohm
    SRF = 71.2 MHz
anti-resonance: |Z| peaks at 52.5 MHz (0.47 Ohm)
saved: emi_compass_cap.png
```

「コンデンサを足したのにノイズが増えた」の原因になる反共振の山が、
**どの周波数に立つか**を買う前に確認できます。

### 値の書き方

`--c 0.1u` / `--esl 0.5n` / `--c2 4700p` のように、`p` (ピコ) `n` (ナノ) `u` (マイクロ)
`m` `k` `M` `G` の接尾辞が使えます。周波数範囲は `--fmin 10k --fmax 3G` (既定値) で変更可、
出力ファイル名は `-o myplot.png` で指定できます。

### 配線インダクタンスの影響を見る

配線は約 1nH/mm のインダクタンスとして部品の ESL に直列に足されます。
「IC ピンから 5mm 離して置いたら何が起きるか」はこう確かめられます:

```bash
emi-compass cap --c 0.1u --esl 5.5n   # 0.5nH (部品) + 5nH (配線 5mm) の想定
```

SRF が 22.5 MHz → 6.8 MHz に転落し、100 MHz の |Z| が約 11 倍悪化します。
バイパスコンデンサを「ピン直近に置け」と言われる理由が数字で見えます。

### チップビーズの Z/R/X と「目覚めの周波数」

ビーズは高周波で抵抗になってノイズを熱に変える部品です。Z (合成) / R (熱になる分) /
X (ためて返す分) の 3 本線と、R が X を追い越す「目覚めの周波数」を描きます。

```bash
emi-compass bead
```

```
bead: L=350 nH, R_loss=112 Ohm, R_dc=50 mOhm, C_par=1 pF
    |Z| at 100 MHz = 102.6 Ohm
    R/X crossover (wake-up) = 50.9 MHz
```

### 「とりあえずビーズ」の落とし穴 (ビーズ + コンデンサの共振)

ビーズの下流に低 ESR の MLCC を置くと、ビーズの低域インダクタンスと C が共振して、
その周波数では逆にノイズが増えます。

```bash
emi-compass bead --filter-c 10u
```

```
bead + 10 uF (ESR 5 mOhm): resonance ~ 85.1 kHz, peak +10.7 dB at 83.2 kHz
```

「ビーズを入れたらノイズが増えた」の正体が、入れる前に数字で見えます。
`--l` `--r-loss` `--r-dc` `--c-par` でモデルを、`--filter-esr` でダンピングを変えられます。

## モデルと限界 (正直に)

- 実コンデンサは教科書どおりの**直列 RLC** (C + ESL + ESR) として扱います。
  実部品の C・ESR は周波数・温度・DC バイアスで変わるので、このツールの結果は
  **「形と傾向を読む」用途に限ってください**。実設計の最終確認はメーカー提供の
  実測データ (S パラメータ等) が安全です。第 3 本でその読み込みを扱う予定です。
- 挿入損失の計算は 50Ω 測定系の定義 (シャント素子の S21) に従います。
  **実際の電源ラインは 50Ω ではない**ので、データシートの dB がそのまま実機で
  出るわけではありません。`shunt_insertion_loss_db_sys()` に実ラインの
  インピーダンスを与えると、その「しぼみ方」も確認できます
  (例: 0.1μF の 100MHz 挿入損失は 50Ω ベンチで約 38dB、1Ω の電源ラインでは約 11dB)。
- 基板側のインダクタンス (ビア・引き回し) は含みません。上の「配線インダクタンス」の
  ように ESL に上乗せして近似してください。

## 記事の図の再現

連作第 1 本の図 10 枚 (ノイズ 2 モードの説明図、V 字解剖図、容量ファミリー、
並列反共振、三端子構造・比較、挿入損失、50Ω vs 電源ラインのギャップ等) は

共振ピークなどの数値) が生成されます。

## リポジトリ構成

```
src/emi_compass/
  models.py       # 等価回路モデル (コンデンサ=直列 RLC / ビーズ=R_dc + L∥R_loss + 寄生C、挿入損失、共振探索)
  cli.py          # emi-compass コマンド (cap / bead サブコマンド)
scripts/
  make_article1_figures.py   # 連作第 1 本 (コンデンサ編) の図を一括生成
  make_article2_figures.py   # 連作第 2 本 (チップビーズ編) の図を一括生成
tests/
  test_models.py  # 解析値との一致を直接アサート (SRF 22.5MHz、目覚めの周波数、挿入損失の既知値 等) 21 件
```

## テスト

```bash
.venv/bin/python -m pytest tests/
```

「SRF の式 1/(2π√(L·C)) と一致するか」「谷の深さ = ESR か」「右斜面が容量に依存しないか」
「挿入損失が 50Ω 系の既知値と合うか」など、物理量を直接確かめるテストです。
